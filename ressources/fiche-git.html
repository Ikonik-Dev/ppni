🗺️ Le flux Git en un coup d'œil

💡 Comment ça marche ? Git a 4 zones. Tu travailles en local (sur ton ordi) et tu synchronises avec GitHub (sur Internet). Chaque commande déplace tes fichiers d'une zone à l'autre.
📁 Espace de travail Tes fichiers sur l'ordi
git add →
📦 Staging area Fichiers prêts à committer
git commit →
💾 Repo local Historique sur l'ordi
git push →
🌐 GitHub Repo en ligne
⬅️ Dans l'autre sens : git pull pour récupérer les changements depuis GitHub

⌨️ Les 15 commandes à connaître

Commande Catégorie Ce que ça fait Exemple concret
git init Base Initialiser un nouveau repo Git dans le dossier courant À faire une seule fois au début d'un projet
git clone [url] Remote Copier un repo existant depuis GitHub vers ton ordi git clone https://github.com/toi/monprojet.git
git status Base Voir les fichiers modifiés et l'état du repo À lancer avant chaque commit pour vérifier
git add . Base Préparer tous les fichiers modifiés pour le commit Le . signifie "tout le dossier courant"
git add [fichier] Base Préparer un seul fichier pour le commit git add README.md
git commit -m "message" Base Sauvegarder un instantané avec un message descriptif git commit -m "Ajouter section cybersécurité"
git push Remote Envoyer les commits locaux vers GitHub Après un commit, toujours pusher pour sauvegarder en ligne
git pull Remote Récupérer les derniers changements depuis GitHub À faire avant de commencer à travailler en équipe
git log --oneline Base Voir l'historique des commits de façon compacte Chaque ligne = un commit avec son identifiant
git branch Branches Lister toutes les branches locales La branche active est marquée d'un *
git branch [nom] Branches Créer une nouvelle branche git branch amelioration-readme
git checkout [nom] Branches Basculer sur une branche existante git checkout amelioration-readme
git checkout -b [nom] Branches Créer une branche ET basculer dessus en une commande git checkout -b nouvelle-feature
git merge [branche] Branches Fusionner une branche dans la branche courante Depuis main : git merge amelioration-readme
git diff Base Voir exactement ce qui a changé dans les fichiers Les lignes en vert = ajoutées, en rouge = supprimées

✍️ Écrire un bon message de commit

✅ À faire

Messages clairs et précis

  • "Ajouter section cybersécurité au README"
  • "Corriger faute de frappe dans l'introduction"
  • "Créer page d'accueil avec navigation"
  • Verbe à l'infinitif · Court (50 caractères max) · 1 commit = 1 action
❌ À éviter

Messages vagues ou trop généraux

  • "modifications"
  • "ça marche maintenant"
  • "fix"
  • "travail du vendredi"
